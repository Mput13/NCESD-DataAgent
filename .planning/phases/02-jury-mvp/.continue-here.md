<decisions_made>

- Option 2 is the target architecture.
  RAG receives `RetrievalInput` with multiple probes. The raw user query remains one probe
  but is not the only retrieval input.

- Live Qwen owns semantic request understanding.
  Programmatic retrieval code must not encode economic/statistical meaning through
  hardcoded token regexes, stopwords, synonym maps, or share/GDP/inflation special cases.

- Source preferences are explicit user intent.
  If the user says only a source, it is a hard constraint. If the user merely mentions a
  source, it is a soft preference. If no source is mentioned, search all trusted sources.

- Runtime data paths should not depend on `.planning`.
  `.planning` remains durable project memory and agent artifact storage, not the runtime
  manifest/workflow-output directory.
</decisions_made>
