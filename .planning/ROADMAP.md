## Phase 1: Ядро агента — NLU + RAG + поиск данных

**Goal:** Агент принимает NL-запрос, понимает его тип, ищет релевантные датасеты в реестре метаданных Росстата и World Bank, возвращает найденные источники.

**Covers:** NLU-01, NLU-02, NLU-03, NLU-04, SRCH-01, SRCH-02, SRCH-03, SRCH-04

**Deliverables:**
- Индекс метаданных Росстата (metadata.jsonl) для RAG-поиска
- Индекс метаданных World Bank (indicators.json) для RAG-поиска
- Оркестратор на Yandex GPT с OpenAI SDK
- Инструменты: `find_data` (поиск), классификация типа запроса
- Уточняющий диалог для неоднозначных запросов
- Обработка «данных нет»

**UAT:**
- [ ] Простой запрос → система находит релевантный датасет
- [ ] Неоднозначный запрос → система задаёт уточняющий вопрос
- [ ] Запрос без данных → честный ответ «данных нет»

---

## Phase 2: Извлечение данных + генерация артефактов

**Goal:** Агент извлекает данные из Parquet через детерминированный код, формирует определение/дизайн исследования, генерирует скрипт сборки и собирает итоговый датасет.

**Covers:** DATA-01, DATA-02, DATA-03, DATA-04, DATA-05, ART-01, ART-02, ART-03, ART-04, ART-05, ART-06

**Deliverables:**
- Инструмент `get_table` — чтение Parquet через pandas с фильтрацией
- Вычисление производных метрик через сгенерированный код
- Генерация промежуточных артефактов (определение, дизайн, структура)
- Генерация Python-скрипта сборки
- Сборка итогового датасета (CSV/Excel) с метаданными и ссылками
- Каждая цифра — со ссылкой на источник

**UAT:**
- [ ] Простой запрос → end-to-end от NL до CSV с данными
- [ ] Сравнительный запрос → датасет с данными из нескольких стран/периодов
- [ ] Производная метрика → корректное вычисление через код
- [ ] Скрипт сборки скачивается и запускается автономно

---

## Phase 3: UI + устойчивость + полировка

**Goal:** Streamlit-интерфейс с прозрачным «следом» работы, обработка edge cases, README и инструкция по запуску.

**Covers:** RBST-01, RBST-02, RBST-03, RBST-04, UI-01, UI-02, UI-03, UI-04, ENG-01, ENG-02, ENG-03, ENG-04

**Deliverables:**
- Streamlit UI с чат-интерфейсом
- Отображение шагов работы ассистента (какие источники нашёл, какие отверг)
- Скачивание датасетов и скриптов из UI
- Корректная обработка всех 6 типов запросов
- README с описанием архитектуры
- requirements.txt + инструкция по запуску

**UAT:**
- [ ] Все 6 типов запросов обрабатываются корректно через UI
- [ ] «След» работы ассистента виден в интерфейсе
- [ ] Непрограммист может использовать интерфейс
- [ ] `pip install -r requirements.txt && streamlit run app.py` запускает систему
- [ ] Прогон по 5–8 тест-кейсам с количественной оценкой
