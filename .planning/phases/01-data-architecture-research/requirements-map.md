## Embedding and Indexing Evidence Contract

Retrieval evidence must declare how dense retrieval is indexed and queried. If Yandex AI Studio embeddings are used, source-card or chunk documents use a stable document embedding mode such as `text-search-doc`, while natural-language user queries use `text-search-query`. If credentials are absent, Phase 1 evidence must record the provider/model family that would be used, the missing credential, and the credential-aware fallback used for local verification.

Embedding input text must be source-card or chunk metadata only: title, description or methodology summary, source family, dataset/package/resource/indicator ids, units, dimensions, coverage hints, geography hints, provenance URLs/resource URLs, availability flags, quality flags, and rejection-risk notes. Dense retrieval must not embed raw numeric tables, extracted numeric answers, or LLM-generated numeric claims.
