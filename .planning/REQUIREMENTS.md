# Requirements: DataAgent

**Defined:** 2026-05-08
**Core Value:** Опора на факты из реальных данных — каждая цифра со ссылкой, числа из данных извлекает код, не LLM

## v1 Requirements

### Понимание запроса (NLU)

- [x] **NLU-01**: Система принимает запрос на естественном языке (русский) и классифицирует его тип (простой, сравнительный, исследовательский, производная метрика, неоднозначный, нет данных)
- [x] **NLU-02**: Система формализует запрос в определение исследования: география, временные рамки, дисциплинарный ракурс, конкретные вопросы
- [x] **NLU-03**: Для неоднозначных запросов система задаёт уточняющие вопросы и учитывает ответы пользователя
- [x] **NLU-04**: Система корректно определяет, когда запрашиваемых данных нет, и честно сообщает об этом

### Поиск данных (SEARCH)

- [x] **SRCH-01**: RAG-индекс по метаданным Росстата (metadata.jsonl) для поиска релевантных показателей
- [x] **SRCH-02**: RAG-индекс по метаданным World Bank (indicators.json, 29K+ индикаторов)
- [x] **SRCH-03**: Инструмент `find_data` — поиск датасетов по ключевым словам, теме, стране, периоду
- [x] **SRCH-04**: Результаты поиска включают метаданные: источник, период, география, единицы измерения

### Извлечение данных (DATA)

- [x] **DATA-01**: Чтение Parquet-файлов Росстата через детерминированный код (конкретная библиотека не выбрана)
- [x] **DATA-02**: Чтение Parquet-файлов World Bank через детерминированный код (конкретная библиотека не выбрана)
- [x] **DATA-03**: Фильтрация данных по стране, периоду, индикатору
- [x] **DATA-04**: Агрегация и группировка данных по измерениям
- [x] **DATA-05**: Вычисление производных метрик (индексы, нормализация, агрегаторы) через код

### Генерация артефактов (ART)

- [ ] **ART-01**: Определение исследования — формальный документ с географией, рамками, вопросами
- [ ] **ART-02**: Дизайн исследования — гипотезы, измерения, индикаторы, группировки
- [ ] **ART-03**: Структура целевого датасета — колонки, типы, единицы измерения
- [ ] **ART-04**: Python-скрипт сборки датасета (скачиваемый, модифицируемый)
- [ ] **ART-05**: Итоговый датасет в формате CSV/Excel с метаданными и ссылками на источники
- [ ] **ART-06**: Каждая цифра в ответе сопровождается ссылкой на источник данных

### Устойчивость (ROBUST)

- [ ] **RBST-01**: Корректная обработка нечётких запросов — уточняющий диалог
- [ ] **RBST-02**: Корректная обработка заведомо некорректных запросов
- [ ] **RBST-03**: Честный ответ «данных нет» с объяснением, без галлюцинаций
- [x] **RBST-04**: LLM не извлекает числа из таблиц — только детерминированный код

### Интерфейс (UI)

- [ ] **UI-01**: Простой интерфейс на Streamlit/Gradio/web UI/CLI (не мессенджер-бот; конкретный вариант не выбран)
- [ ] **UI-02**: Отображение «следа» работы ассистента — шаги, источники, промежуточные артефакты
- [ ] **UI-03**: Возможность скачивания итогового датасета и скрипта
- [ ] **UI-04**: Интерфейс доступен для пользователя-непрограммиста

### Инженерия (ENG)

- [ ] **ENG-01**: Воспроизводимость: README, requirements.txt, инструкция по запуску
- [ ] **ENG-02**: Чистое разделение ответственности: LLM (понимание, маршрутизация) vs код (данные)
- [ ] **ENG-03**: Архитектура допускает лёгкое добавление нового источника данных
- [ ] **ENG-04**: Описание архитектуры в README

## v2 Requirements

### Расширенные источники

- **EXT-01**: Интеграция с НЦСЭД API (CKAN) для поиска по полному каталогу
- **EXT-02**: Подключение дампов ВТО, ЮНЕСКО, ЮНИДО
- **EXT-03**: Web Search для обнаружения дополнительных открытых источников

### Расширенный UX

- **UX-01**: Принятие правок пользователя и обновление результата
- **UX-02**: Журнал сессий с воспроизведением
- **UX-03**: Библиотека образцов исследований (расширяемая)
- **UX-04**: Визуализация данных (графики, таблицы)

| Проприетарные библиотеки | Запрет ТЗ |
| Авторизация пользователей | Прототип, не production |
| Мобильное приложение | Прототип, не production |
